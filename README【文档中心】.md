## 💬 文档使用建议

### ✅ 推荐做法

- **使用 Ctrl+F 搜索**：在文档中快速定位内容
- **标记重点**：在 IDE 中标记高风险函数和优化建议
- **定期更新**：每个迭代完成后更新评估表
- **分享知识**：在团队会议中分享关键发现

### ❌ 避免做法

- 不要跳过"核心流程完整性"章节，这是理解项目的基础
- 不要忽视"高风险函数"的隐患，这些是 Bug 的主要来源
- 不要延迟 P0 级优化，这些会影响系统稳定性
- 不要在没有测试的情况下修改高难度函数
